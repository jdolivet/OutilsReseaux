Aide sur Ping

Le Ping permet de vérifier si un hôte est en ligne. Remplissez le formulaire comme suit:

Hôte:

Le nom de la machine à tester (www.fifi.org par exemple) ou une adresse IP telle que 127.0.0.1.

Nombre de paquets:

Le nombre de paquets à envoyer à l'hôte. Si plus d'un paquet est envoyé, vous pourrez obtenir des statistiques telles que les temps les plus courts et les plus longs qu'a pris l'hôte à répondre.

Délai entre les paquets:

Le temps en secondes après chaque réponse avant d'envoyer un autre paquet.

Formater le résultat:

Vous pouvez choisir de voir les résultats bruts, ou de les formater dans un tableau récapitulatif.

Si vous avez opté pour un résultat formaté, le tableau récapitulatif contient les colonnes suivantes:

Numéro de séquence:

Chaque paquet envoyé contient un numéro de séquence, et le tableau liste les paquets dans l'ordre où ils sont reçus. Si un hôte est très loin (en termes de distance dans le réseau Internet), ou si un routeur entre Fifi.Org et l'hôte est surchargé, les réponses peuvent revenir dans un ordre différent (ce qui n'est généralement pas un bon signe).

Réponse venant de:

L'adresse IP de la machine qui a répondu au ping. Elle peut être différente de l'adresse de l'hôte pingé si la machine est derrière un pare-feu (firewall).

Durée de vie:

Le nombre de passerelles ou de routeurs que le paquet est autorisé à traverser avant d'être détruit (ceci afin d'éviter les boucles infinies). Dans le cas général, quand un hôte répond à un ping, il utilise en temps de vie de 255 pour la réponse (64 est aussi couramment utilisé).

Le nombre de routeurs traversés par la réponse peut donc être obtenu en soustrayant le temps de vie à l'arrivée de 255 (ou 64). Si le temps de vie est moins que 64, vous pouvez utiliser 64 comme base car les paquets traversant quelques 200 routeurs avant d'arriver ici sont plutôt rares.

Vous pouvez utiliser traceroute pour obtenir la liste de ces routeurs.

Délai:

Le temps pris pour faire l'aller-retour de Fifi.Org à l'hôte, en millisecondes.

Statistiques (paquets envoyés, reçus, pertes; délais minimum, moyen et maximum):

Sans commentaires, devrait être explicite.
